# 银行服务聊天机器人 - 数据库设计说明

## 概述

本项目采用微服务架构，包含三个核心服务，每个服务都有独立的数据库。本文档详细说明了每个服务的数据库表结构设计。

## 1. 用户服务 (User Service)

**数据库名**: user_service

### 表结构说明

#### users (用户表)
存储系统用户的基本信息，包括登录凭证和个人资料。

#### roles (角色表)
定义系统中的角色类型，如普通用户、VIP用户、管理员等。

#### user_roles (用户角色关联表)
建立用户与角色的多对多关系。

## 2. 银行服务 (Bank Service)

**数据库名**: bank_service

### 表结构说明

#### accounts (银行账户表)
存储用户的银行账户信息，包括账户类型、余额、状态等。

#### transactions (交易记录表)
记录所有账户的交易历史，包括存款、取款、转账等操作。

#### products (银行产品表)
存储银行提供的各类产品信息，如贷款、存款、保险、基金等。

#### user_products (用户产品关联表)
记录用户购买的银行产品信息。

## 3. 聊天机器人服务 (Chatbot Service)

**数据库名**: chatbot_service

### 表结构说明

#### chat_sessions (聊天会话表)
记录用户与聊天机器人的会话信息。

#### chat_messages (聊天消息表)
存储会话中的具体消息内容，区分用户消息和机器人回复。

#### faq (常见问题表)
维护常见问题及其标准答案，支持分类和关键词搜索。

#### intents (意图识别表)
定义用户可能的意图类型，用于自然语言处理。

#### entities (实体识别表)
定义意图相关的实体信息，辅助语义理解。

#### chatbot_config (聊天机器人配置表)
存储聊天机器人的配置参数。

## 设计原则

1. **微服务独立性**: 每个服务拥有独立的数据库，确保服务间的解耦。
2. **逻辑删除**: 所有表都包含deleted字段，支持软删除。
3. **时间戳**: 所有表都包含created_at和updated_at字段，便于审计和追踪。
4. **索引优化**: 为常用查询字段创建索引，提高查询性能。
5. **数据完整性**: 使用外键约束保证数据一致性。

## 服务间通信

虽然每个服务有独立的数据库，但服务间可以通过REST API或消息队列进行通信，实现数据共享和业务协同。
